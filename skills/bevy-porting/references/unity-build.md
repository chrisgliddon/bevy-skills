# bevy-porting — Build Settings + PlayerSettings → Cargo + features + targets

> Referenced from `bevy-porting/SKILL.md § Unity (priority)`.

## High-level mapping

Unity's **Build Settings** (platform list, scenes, scripting backend) and **PlayerSettings** (icons, bundle ID, splash, signing) live in a single `BuildProfile` asset. Bevy has no equivalent monolithic file — configuration is distributed:

| Unity concept | Bevy equivalent |
|---|---|
| Build Settings — platform | `cargo build --target <triple>` |
| Build Settings — scenes list | No equivalent; `App::run()` + `Startup` systems |
| Scripting backend (Mono / IL2CPP) | Not applicable — Rust compiles natively via `rustc` |
| PlayerSettings — Bundle ID | `Info.plist` (macOS/iOS), `AndroidManifest.xml`, or `[package.metadata.bundle]` |
| PlayerSettings — icons | Platform-specific (`icns`, `ico`, `.rc`, Play Store PNG) |
| Asset Bundles | No core equivalent; ship an `assets/` folder or `include_bytes!` for small assets |
| Build Profile | Split across `Cargo.toml`, profile overrides, and CI scripts |

## Target platform table

| Unity target | Cargo target triple | Tooling |
|---|---|---|
| Windows Standalone | `x86_64-pc-windows-msvc` | `cargo build --release --target …` |
| macOS Standalone | `aarch64-apple-darwin` / `x86_64-apple-darwin` | `cargo build --release --target …` |
| Linux Standalone | `x86_64-unknown-linux-gnu` | `cargo build --release --target …` |
| WebGL | `wasm32-unknown-unknown` | See `bevy-wasm-webgpu` |
| Android | `aarch64-linux-android` | `cargo-mobile2` or `xbuild` |
| iOS | `aarch64-apple-ios` | `cargo-mobile2` |
| PS5 / Xbox / Switch | — | Not officially supported in Bevy 0.18 |

For a universal macOS binary: `cargo build --target x86_64-apple-darwin && cargo build --target aarch64-apple-darwin` then `lipo -create` the two outputs.

## Feature trimming

Feature selection is the biggest single lever on binary and bundle size. See `bevy-cargo-features` for the full table. Condensed:

```toml
# Trimmed release build — start here, add back what you need.
[dependencies]
bevy = { version = "0.18", default-features = false, features = [
    "3d_api",          # component types, no default renderer
    "bevy_winit",      # window + event loop
    "mouse", "keyboard",
] }

[profile.release]
opt-level = 3
lto = "fat"
codegen-units = 1
strip = "debuginfo"
```

`default-features = false` drops ~30 features. For a desktop game you usually re-add `"3d"` (or `"2d"`) instead of `"3d_api"` — the `_api` variants are for custom renderers.

## WebGL → WASM

| Unity WebGL | Bevy WASM |
|---|---|
| `.wasm` + `.data` blob + JS loader | `.wasm` + `wasm-bindgen` JS glue + `assets/` folder |
| Assets bundled into `.data` at build time | Assets served as plain files from the web server |
| IL2CPP compiles C# to C++ | `rustc` compiles to native WASM — no intermediate step |

```bash
cargo build --release --target wasm32-unknown-unknown
wasm-bindgen --target web --out-dir public/ \
    target/wasm32-unknown-unknown/release/mygame.wasm
wasm-opt -Oz public/mygame_bg.wasm -o public/mygame_bg.wasm   # binaryen
```

`wasm-opt -Oz` from the `binaryen` toolkit is the WASM equivalent of IL2CPP stripping — routinely shaves 20–40% from the binary. Cross-link: `bevy-wasm-webgpu` covers the full pipeline, backend selection (`webgl2` vs `webgpu`), and the `assets/` path gotcha.

## Bundle ID, signing, icons

### macOS / iOS

```
# cargo-bundle reads [package.metadata.bundle] from Cargo.toml
[package.metadata.bundle]
name            = "My Game"
identifier      = "com.example.mygame"
icon            = ["assets/icons/512x512.png"]
```

`cargo install cargo-bundle && cargo bundle --release` produces a `.app`. For iOS distribution use `cargo-mobile2` (`cargo install cargo-mobile2`), which generates an Xcode project, handles code signing, and supports `cargo mobile ios build`.

### Android

```xml
<!-- android/app/src/main/AndroidManifest.xml (generated by cargo-mobile2) -->
<manifest package="com.example.mygame">
  <application android:label="My Game" android:icon="@mipmap/ic_launcher">
    <activity android:name=".MainActivity" android:exported="true" />
  </application>
</manifest>
```

Sign with a `.keystore` file via `apksigner` or Android Studio. `cargo-mobile2` scaffolds this automatically.

### Windows

Add `app.rc` with an `ICON` resource and pass it to your build via `build.rs`. For an MSI installer: `cargo install cargo-wix && cargo wix`.

## Strip and minify

```bash
# Native: strip debug symbols after cargo build --release
strip target/release/mygame                   # Linux / macOS
# Or use the Cargo profile above — strip = "debuginfo"

# Measure bloat:
cargo install cargo-bloat
cargo bloat --release --crates

# WASM: run after wasm-bindgen
wasm-opt -Oz out.wasm -o out.wasm
```

`cargo build --release` alone typically cuts binary size by 4–10× vs debug. `strip` removes another 30–60%. For WASM, `wasm-opt -Oz` is the equivalent of Unity's IL2CPP byte-code stripping.

## Asset bundling

Unity AssetBundles have no direct Bevy equivalent. Two patterns:

1. **Ship `assets/` alongside the binary.** Bevy's `AssetServer` loads from the filesystem (native) or a web path (WASM) by default. See `unity-assets.md` for the `AssetServer` loading model.
2. **Embed small assets with `include_bytes!`.** Use a custom `AssetReader` to serve bytes from a static slice. Practical for critical data (shaders, fonts); impractical for large audio or texture packs.

## Gotchas

- **`cargo-mobile2` and `xbuild` are community projects.** `cargo-mobile2` is the more active fork (as of 2025) of the original `cargo-mobile`. `xbuild` is an alternative but less widely maintained — verify current status before depending on it.
- **`cargo-bundle` does not handle iOS signing or notarization.** Use `cargo-mobile2` for iOS end-to-end.
- **Universal macOS binaries need `lipo`.** `cargo` doesn't produce fat binaries natively; you must build each arch separately and merge.
- **`wasm-opt` is not a Cargo plugin** — install `binaryen` from your system package manager or <https://github.com/WebAssembly/binaryen/releases>.
- **Console support (PS5/Xbox/Switch) is absent from Bevy 0.18.** No official renderer backend exists. Community forks exist but are not covered here.
