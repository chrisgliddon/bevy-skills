# Bevy 0.18 — Custom Material Reference

Writing a custom `Material` with `AsBindGroup`. See also
[lighting](lighting.md) and [mesh-primitives](mesh-primitives.md).

---

## Complete Example: DissolveMaterial

The following snippet is verified to compile against Bevy 0.18.

```rust
use bevy::asset::{Asset, AssetPath};
use bevy::pbr::{Material, MaterialPlugin};
use bevy::prelude::*;
use bevy::reflect::TypePath;
use bevy::render::render_resource::AsBindGroup;
use bevy::shader::ShaderRef;

#[derive(Asset, AsBindGroup, TypePath, Debug, Clone)]
struct DissolveMaterial {
    #[uniform(0)]
    progress: f32,
    #[texture(1)]
    #[sampler(2)]
    noise: Handle<Image>,
}

impl Material for DissolveMaterial {
    // 0.18: shader paths via ShaderRef::Path or ShaderRef::Handle
    fn fragment_shader() -> ShaderRef {
        ShaderRef::Path(AssetPath::from("shaders/dissolve.wgsl"))
    }

    // 0.18: prepass/shadow config moved from plugin fields to trait methods.
    // Override only when you need to opt out — both default to true.
    fn enable_prepass() -> bool { false }
}

fn main() {
    App::new()
        .add_plugins(DefaultPlugins)
        .add_plugins(MaterialPlugin::<DissolveMaterial>::default())
        .add_systems(Startup, setup)
        .run();
}

fn setup(
    mut commands: Commands,
    mut meshes: ResMut<Assets<Mesh>>,
    mut materials: ResMut<Assets<DissolveMaterial>>,
    asset_server: Res<AssetServer>,
) {
    let mesh = meshes.add(Cuboid::new(1.0, 1.0, 1.0));
    let material = materials.add(DissolveMaterial {
        progress: 0.0,
        noise: asset_server.load("textures/noise.png"),
    });

    commands.spawn((
        Mesh3d(mesh),
        MeshMaterial3d(material),
        Transform::from_xyz(0.0, 0.5, 0.0),
    ));
}
```

## Material Trait Methods

Override only the methods you need. All have sensible defaults.

```rust
impl Material for MyMaterial {
    // Required if you have a vertex shader:
    fn vertex_shader() -> ShaderRef {
        ShaderRef::Path(AssetPath::from("shaders/my.vert.wgsl"))
    }

    // Required if you have a fragment shader:
    fn fragment_shader() -> ShaderRef {
        ShaderRef::Path(AssetPath::from("shaders/my.frag.wgsl"))
    }

    // Controls whether this material runs in the depth/normal prepass.
    // Default: true. Disable for materials that modify vertex positions
    // in ways that break the prepass depth buffer.
    fn enable_prepass() -> bool { true }

    // Controls whether this material casts shadows.
    // Default: true.
    fn enable_shadows() -> bool { true }

    // Controls the blend mode / alpha handling.
    // Default: AlphaMode::Opaque
    fn alpha_mode(&self) -> AlphaMode {
        AlphaMode::Blend
    }

    // Low-level: specialise the render pipeline (e.g. to change primitive topology).
    // Use `SpecializedMeshPipelineError` from bevy::pbr.
    // fn specialize(...) -> Result<...> { ... }
}
```

`ShaderRef` variants:
- `ShaderRef::Path(AssetPath::from("shaders/foo.wgsl"))` — loaded from assets folder.
- `ShaderRef::Handle(handle)` — pre-loaded `Handle<Shader>`.
- `ShaderRef::Default` — use the engine's built-in PBR shader.

## AsBindGroup Derive Attributes

The derive macro reads attributes to generate GPU binding layout.

```rust
#[derive(Asset, AsBindGroup, TypePath, Debug, Clone)]
struct MyMaterial {
    // Binding 0: a uniform buffer containing a single f32.
    // The whole struct at binding N can also hold compound types if they are
    // ShaderType (use bevy::render::render_resource::ShaderType).
    #[uniform(0)]
    progress: f32,

    // Binding 1: a texture (2D by default).
    // Binding 2: its sampler.
    // The pair must use consecutive indices.
    #[texture(1)]
    #[sampler(2)]
    noise: Handle<Image>,

    // For a texture array use:
    // #[texture(3, dimension = "2d_array")]
    // #[sampler(4)]
    // layers: Handle<Image>,
}
```

**Binding index convention:** Start at 0. Bindings 0–N are your material bindings;
the engine's own PBR bindings sit in a separate bind group so there is no conflict.
Consecutive indices are not required, but gaps waste bind group entries.

## 0.18-Specific: AsBindGroup::label()

In 0.18, `AsBindGroup::label()` is **required** if you hand-roll the `impl AsBindGroup`.
The `#[derive(AsBindGroup)]` macro generates it automatically from the type name:

```rust
// Generated by the derive:
fn label() -> &'static str {
    "MyMaterial"
}
```

If you implement `AsBindGroup` manually (rare), you must provide this:

```rust
impl AsBindGroup for MyMaterial {
    fn label() -> &'static str { "MyMaterial" }
    // ... rest of impl
}
```

## Pitfalls

- **Forgetting `MaterialPlugin::<M>::default()`** — without the plugin, the renderer
  never registers the material type and nothing will render (no error, just black).

- **Name collision with the prelude's `Material`** — `bevy::prelude::*` does not
  re-export `Material`. Import it as `use bevy::pbr::Material;`. If you see
  "multiple candidates" errors, qualify the path explicitly.

- **`MaterialPlugin` field removal** — in 0.18, `MaterialPlugin::<M> { prepass_enabled, shadows_enabled, ..default() }` no longer compiles. Those fields moved to trait methods (`enable_prepass()`, `enable_shadows()`).

- **`#[uniform(N)]` and `ShaderType`** — only types that implement `ShaderType`
  (from `bevy::render::render_resource`) can be used with `#[uniform]`. Primitives
  `f32`, `Vec2`, `Vec3`, `Vec4`, `Color`, `Mat4` all work. Plain Rust structs need
  `#[derive(ShaderType)]`.

- **Vertex shader required for vertex animation** — if your WGSL modifies vertex
  positions, also override `vertex_shader()` and mark `enable_prepass() -> false`
  to prevent the prepass from rendering with the wrong depth.

- **Draw functions are per-phase in 0.18.** The old single `MaterialDrawFunction`
  split into `MainPassOpaqueDrawFunction`, `MainPassAlphaMaskDrawFunction`,
  `MainPassTransparentDrawFunction`, and `PrepassOpaqueDrawFunction` /
  `PrepassAlphaMaskDrawFunction`. If you're registering a draw function manually
  (rare — `MaterialPlugin` handles this), register against the phase you actually
  render in. Mixing them silently renders the material in the wrong pass.

- **`BindGroupLayout::create(...)` was removed.** Build layouts via
  `BindGroupLayoutDescriptor::new(...)` (or the `#[derive(AsBindGroup)]` macro,
  which is the recommended path) and resolve through
  `pipeline_cache.get_bind_group_layout(&desc)`. The old
  `device.create_bind_group_layout(&entries)` shape no longer compiles.
